Portfolio V3

During my work at Uni Bonn, I helped port a vanilla KoaJS backend to a headless Strapi CMS instance. That gave me the idea to create the next iteration of my portfolio website, dubbed Portfolio V3, in the same way. My previous two portfolio websites used Angular and React, with all the content manually written in. As a small website with little content, it makes sense to just generate a static website and host it somewhere, such as Github Pages. However, this also meant adding content manually as well. What if instead I stored all my content in a CMS, loaded them in the frontend, and then ran the static site generation? Thus came about this portfolio website, which also includes this simple case study that you're reading.

Since I am also taking this opportunity to learn something new while I am at it, I opted for Statamic, a CMS based on Laravel and PHP. Statamic provides its own frontend mechanisms, along with static site generation, however it can also be run headless through REST as well as GraphQL. In order to make data fetching easier, I used urql, a library that queries Statamic's GraphQL API. I can easily write up the query in a single object, which includes the title, content, and related media, without having to worry about parameters in a typical REST API call. Although it requires a Pro license in order to run a live site with the REST and GraphQL APIs enabled, since I am not hosting Statamic itself, I can just run the frontend static site generator and host the built site. Statamic Pro has an infinite trial as long as it's during development, so I can get the benefits for my unique use case.

Since the previous websites used Angular and React, and I already use Vue at work, this time I opted for a Svelte frontend. More specifically, SvelteKit, which comes with its own file based route mechanism. Svelte separates routes into files and folders, where each folder is a route. Every route has a +page.svelte file for rendering, and +layout.svelte file for common components to be rendered in multiple routes. For client side JavaScript (or TypeScript), a +page.ts file can be included, and +page.server.ts for server side functionality. Using the adapter for static site generation, the functions I put in +page.server.ts will load before being rendered statically.

To style the website, I used BulmaCSS. Of course, using a TailwindCSS library might have made it a lot faster, since that's what I am used to, but I wanted to see how Bulma also works, and it even comes with a small component library of its own, which covers the use case for a portfolio website like mine.

With all that out of the way, the hardest part is just creating the content itself. I intend to showcase my skills, projects, and case studies like this on the website. Skills would be categorized under Web, Apps, Games, General, and Embedded, and could be a Language (like JavaScript), a Framework (like Svelte), or a Tool (like Zed or VSCode). Projects are related to the skills required for them, and a project could involve one or more case studies. The biggest benefit is that now I can spend most of the time on the content, run the static generator, and the portfolio website would be updated!
